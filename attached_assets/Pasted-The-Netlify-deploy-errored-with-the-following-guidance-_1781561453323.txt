Diagnosis
- pnpm is aborting the install because the repository's pnpm-lock.yaml doesn't match the current overrides configuration in package.json (pnpm's ERR_PNPM_LOCKFILE_CONFIG_MISMATCH). Netlify runs a frozen install, so the mismatch causes the dependency installation step to fail.

Solution
1. Verify your overrides in package.json (are they intentional)
   - Open package.json in the repo to confirm any "overrides" entries are correct: https://github.com/emomba/CineClub/blob/main/package.json

2. Regenerate and commit an updated lockfile (recommended)
   - On your local machine (repo root), run:
```bash
pnpm install --no-frozen-lockfile
git add pnpm-lock.yaml
git commit -m "Regenerate pnpm-lock.yaml to match package.json overrides"
git push
```
   - This updates pnpm-lock.yaml so CI can perform a frozen install successfully.

3. Alternative: change the install step in CI (if you cannot commit the lockfile)
   - In Netlify build settings, run a non-frozen install before the build, e.g. set the install command to:
```bash
pnpm install --no-frozen-lockfile
```
   - Or prepend that in your custom build command in Netlify (not recommended long-term — better to commit the lockfile).

Notes
- The logs also show a pnpm version adjustment ([line 17](#L17) / [line 19](#L19)). Make sure you regenerate the lockfile with a pnpm version compatible with your CI (Netlify is using pnpm 9.x here). If you need to change Node versions in Netlify, see: https://docs.netlify.com/configure-builds/manage-dependencies/#node-js-and-javascript
